# 노바의 PRD 리뷰 기준

PM들이 PRD 제출 전 자가 점검, AI 사전 검토 시 활용하는 체크리스트.

---

## 필수 검토 항목 (노바가 가장 먼저 보는 것)

### 1. 문제 정의

**"어떤 문제를 해결하려고 하는가?"**

체크포인트:
- 문제가 구체적으로 정의되어 있는가 (막연한 "개선" 아닌, 누구의 어떤 고통인지)
- 문제의 크기/심각도가 근거와 함께 제시되는가 (데이터, 유저 인용, 비즈니스 임팩트)
- 이 문제를 지금 풀어야 하는 이유가 있는가 (왜 다른 문제보다 먼저?)
- 문제와 솔루션이 논리적으로 연결되는가 (이 솔루션이 정말 이 문제를 해결하는지)

나쁜 예: "UX를 개선한다", "기능을 추가한다"
좋은 예: "Research AX 사용자가 설문 설계서를 보면 무슨 말인지 모를 가능성이 높다. 하지만 이 과정을 생략하면 품질이 떨어진다."

### 2. 유저 여정 (User Journey)

**"유저가 들어왔을 때 어떤 프로세스를 거쳐서 이 경험을 하게 되는가?"**

체크포인트:
- 유저의 진입 지점부터 최종 결과물까지 단계별 흐름이 그려져 있는가
- 각 단계에서 유저가 보는 화면/선택지가 구체적으로 기술되어 있는가
- 세그먼트별로 여정이 다른 경우, 각각 따로 기술되어 있는가
- 이탈할 수 있는 지점과 그에 대한 대응이 있는가
- 시간 감각이 있는가 (각 단계에 얼마나 걸리는지)

나쁜 예: "유저가 설문을 생성하고 결과를 본다"
좋은 예: "질문 입력 → 맥락 파악 2-3개 답변 → '완료되면 알림받기' → 2분 후 알림 → 설문 미리보기 → '이대로 진행' → 합성 패널로 47초 후 결과. 총 소요시간 약 5분."

## 추가 검토 항목 (위 두 가지가 통과된 후)

- [ ] 전략 정합성: 노바의 비전 문서(설계안, 프라이싱)와 충돌 없는가
- [ ] 성공 지표: 킬/고 판정이 가능한 정량 기준이 있는가
- [ ] 리스크: 실패 시나리오와 대응 계획이 있는가
- [ ] 세그먼트 고려: Researcher Pro / Research AX / SMB별 경험 차이가 반영되었는가
