Lesson 1

Google Calendar 연결

PM-OS의 핵심은 내 업무 도구가 연결되어 있다는 것입니다. 캘린더를 연결하면 /daily-plan이 오늘 미팅을 자동으로 가져옵니다.

claude mcp add google-calendar -- npx -y @anthropic-ai/google-calendar-mcp@latest copy

브라우저에서 Google 인증을 완료하세요.

확인 방법

  • Claude Code에서 "오늘 일정 보여줘" 입력
  • 캘린더 데이터가 나오면 성공
Lesson 2

Slack 연결

Slack을 연결하면 /slack-message로 작성한 메시지를 채널에 바로 보낼 수 있습니다.

Claude.ai > Settings > Integrations에서 Slack 연결하거나:

/connect-mcps connect to slack copy

확인 방법

  • "Slack 채널 목록 보여줘" 입력
  • 채널 리스트가 나오면 성공
연결이 안 되면 건너뛰세요. 나중에 /connect-mcps로 언제든 추가할 수 있습니다.
Lesson 3

Atlassian (Jira/Confluence) 연결

Jira를 연결하면 /status-update에 스프린트 데이터가 자동 반영되고, /create-tickets로 티켓을 직접 만들 수 있습니다.

Claude.ai > Settings > Integrations에서 Atlassian 연결하거나:

/connect-mcps connect to atlassian copy

확인 방법

  • "Jira 프로젝트 목록 보여줘" 입력
  • 프로젝트 리스트가 나오면 성공
실습 1

/daily-plan 실행

MCP가 연결됐으니, 진짜 데이터로 오늘의 플랜을 만들어봅니다.

/daily-plan copy

기대 결과

  • 오늘 캘린더에서 미팅을 자동으로 가져옴
  • 미팅별 컨텍스트 (참석자, 안건 추정)
  • 우선순위가 정리된 오늘의 Top 3
  • Jira에서 할당된 티켓 참조 (연결된 경우)
MCP 없이도 동작하지만, 캘린더 데이터가 있을 때와 없을 때의 차이를 느껴보세요. 이게 PM-OS가 ChatGPT와 다른 이유입니다.
실습 2

/meeting-notes 실행

오늘 미팅이 끝나면 (또는 어제 미팅 노트가 있다면) 바로 써보세요.

/meeting-notes copy

미팅 노트를 붙여넣으세요. 음성 메모 텍스트, 자유 형식 메모, 녹음 전사 등 아무거나 OK.

기대 결과

  • 구조화된 미팅 노트 (요약, 결정사항, 액션아이템)
  • 각 액션아이템에 오너와 기한 자동 배정
  • outputs/meeting-notes/에 파일 저장
미팅 직후에 바로 쓰는 게 가장 좋습니다. 기억이 생생할 때.
Day 1 요약
MCP 연결 = PM-OS가 내 실제 데이터를 읽을 수 있게 하는 것
/daily-plan = 매일 아침 루틴. 캘린더 + Jira 기반 우선순위
/meeting-notes = 미팅 후 루틴. raw 노트에서 구조화된 결과물
