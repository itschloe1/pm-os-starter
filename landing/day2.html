실습 1

/slack-message

팀에 공유할 내용이 있을 때 씁니다. 채널과 톤에 맞게 메시지를 만들어주고, Slack MCP가 연결되어 있으면 바로 전송도 됩니다.

/slack-message copy

상황을 설명하세요. 예: "어제 플랜 개편 미팅에서 결정된 사항을 제품팀 채널에 공유하고 싶어"

기대 결과

  • 채널과 톤에 맞는 Slack 메시지 초안
  • Slack MCP 연결 시 바로 전송 가능
  • outputs/slack-messages/에 저장
"이 메시지를 #product-team 채널에 보내줘"라고 하면 Slack MCP가 직접 전송합니다.
실습 2

/status-update

스테이크홀더에게 보내는 주간보고. Jira 데이터를 자동으로 반영합니다.

/status-update copy

이번 주에 한 일을 간단히 나열하거나, 그냥 실행만 해도 Jira + 미팅 노트에서 자동으로 뽑아줍니다.

기대 결과

  • "이번 주 완료 / 진행 중 / 다음 주 계획" 구조
  • Jira 스프린트에서 티켓 상태 자동 반영
  • 읽는 사람(경영진, 팀)에 맞게 톤 조절
금요일 오후에 /status-update를 루틴으로 쓰면 주간보고가 10분 안에 끝납니다.
실습 3

/action-items

미팅 노트나 Slack 스레드에서 액션아이템만 쏙 뽑아줍니다.

/action-items copy

미팅 노트나 Slack 스레드를 붙여넣으세요. 어제 /meeting-notes로 만든 노트를 넣어봐도 좋습니다.

기대 결과

  • 토픽별로 그룹된 체크리스트
  • 각 항목에 오너, 기한 자동 배정
  • Jira 연동 시 티켓 생성 제안
Day 1 + Day 2 조합의 위력
